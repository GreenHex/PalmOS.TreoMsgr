What is TreoMsgr?

With TreoMsgr, you can schedule messages for delivery in the future. From v4.5 onwards, TreoMsgr supports Groups Messaging. A Plug-in for 2day (see: www.shsh.com) is also included.

Installing TreoMsgr

HotSync or copy TreoMsgr.PRC to your Treo phone. It will appear in the Treo Launcher. It should be installed in the internal memory for proper functioning.

Uninstalling TreoMsgr

Disable TreoMsgr and delete from the [Home]->[Menu]->[Delete] window.

Compatibility

TreoMsgr will work only on all current Treo series Smartphones. It has been developed on a Cingular Treo 650 running the latest updater available on the Palm website.

Operation

Invoking TreoMsgr from the Preferences Panel will display the TreoMsgr Preferences form. It has only one option to enable or disable TreoMsgr. You can create and modify messages from within TreoMsgr. TreoMsgr also maintains a detailed log of message deliveries. In the "View Scheduled Messages" form, you can tap any displayed row to bring-up additional editing options.

From v4.5, TreoMsgr supports Groups Messaging. You can use the inbuilt groups form to define groups, or you can enter individual recipients in the "To: " field of the Tasks form. You cannot combine defined groups with individual numbers. The maximum number of recipients is 10. Numbers can be in both international (+010123456789) or local (0123456789) formats. The recipients addresses must be separated by a Comma and a Space. i.e. +0123456789, +0123456789, etc.

The messages can also be managed in the Treo native Tasks application.

TreoMsgr will recognise and schedule only the following tasks:
  • Tasks in category "TreoMsgr"
  • Having a valid date and time in the future.
  • TreoMsgr stores its information in the Task Notes in the following format:
    <tm>
<tel>0123456789</tel>
<msg>Hello! This is my message.</msg>
</tm>
    The "tags" in RED must appear exactly as shown. The telephone number appears between <tel> and </tel> tags and the message between <msg> and </msg> tags.

  • The "Alarm: _ days earlier" is ignored.
  • Tasks must be edited with the inbuilt Tasks application, TreoMsgr or 2day (see: www.shsh.com) for everything to work as expected.

Features of TreoMsgr

  • Small PRC (~ 73 KB).
  • Integrated with the Treo application environment. Uses the inbuilt Tasks (ToDo) database. No separate DBs, conduits, etc. for you to manage.
  • You can use all of Tasks features, especially the complex "repeat" functionality to schedule messages.
  • From v4.5, full support for messaging groups upto 10 recipients.

Limitations of TreoMsgr

  • It is not possible to combine "Groups" with individual phone numbers in the "To: " field.
  • Recursive groups not (yet) implemented.

Known Bugs and Issues

  • [NORMAL] Conduit errors while HotSyncing with Palm Desktop. This occurs because TreoMsgr removes deleted task entries instead of marking them as "deleted" (as recommended in the developer documentation).
  • [RARE] White screen freeze and buttons not responding under certain conditions, after despatching messages to oneself while in TreoMsgr. Cause presently unknown. A warm-reset should clear the error.
  • [NORMAL] TreoMsgr will not send messages to email addresses on the Treo 680. Cause unknown.
